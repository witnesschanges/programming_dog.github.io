如何在windows操作系统下创建一个新的hexo博客

2020-04-05

摘要

既然要通过博客来记录自己的技术成长，当然要对如何用hexo创建一个博客来做一个实战记录。防止遗忘同时更加熟悉如何编写一篇博客。

需要用到的工具

仓库： github，当然它对应的版本控制系统git也要安装在自己的电脑上
框架： hexo
操作界面： cmd控制台
编写语言： markdown

工作流程

具体步骤：
第一步：进入到自己创建的本地blog文件夹。
第二步：启动hexo，这一步会创建一个本地的网络连接：
http://localhost:4000
第三步：为自己的博客创建一个名字。
第四步：找到对应的.md文件，使用markdown编辑器进行博客编写。
第五步：清理文件，暂时不知道这一步是干什么的。
第六步：生成文件，为部署到远端做准备。
第七步：部署到远端，这里我用了自己的GitHub域址。

如果要更改某篇博客的一些细节或内容，秩序执行第四步至第七步。另外，博客中文件的插入方式请参见以下网址的第一个高赞回答：
https://www.zhihu.com/question/21065229
如若想要了解怎么搭建hexo博客环境，请学习codesheep的视频：
https://www.bilibili.com/video/BV1Yb411a7ty?t=1165

如何添加实现所有文章的功能模块

因为一开始点所有文章会显示功能不能用，然后会显示以下的图片。

上面提醒有三步。
一、查node的版本，在命令行输入”node -v”命令就行
二、”npm i hexo-generator-json-content –save”。命令行输入时会卡住，但是后来没安装成功也能使用。
三、输入下面一段文字在_config.yml文件里面
jsonContent:
meta: false
pages: false
posts:
title: true
date: true
path: true
text: false
raw: false
content: false
slug: false
updated: false
comments: false
link: false
permalink: false
excerpt: false
categories: false
tags: true

注意！第一行前无空格，然后面一段空两格，在后一段空三格。这个错误不容易发现，花费了很长的时间，总以为是第二步出了问题，其实不然，后面证实第二步不执行也能成功。

如何让图片能正常显示

之前通过将图片部署到github上面，会出现github和hexo博客都不能显示图片的问题，因此问了百度，查到的方法如下。
链接： https://blog.csdn.net/Fitz1318/article/details/86548129

设置站点配置_config.yml:将post_asset_folder: false改为post_asset_folder: true
安装插件:npm install https://github.com/CodeFalling/hexo-asset-image -- save
运行hexo n “XXXXXX”,生成XXXXX.md博文时就会在/source/_posts目录下生成XXXXXX的文件夹，将你想在XXXXX博文中插入的照片放置到这个同名文件夹中即可，图片的命名随意。
添加图片:在想添加的位置写入![](图片名字.图片格式),例如![](1.png)。

实践：
安装插件时可能会出现err，不知道什么原因，不过后面还是能使用。
可以不使用hexo n “XXXXXX”创建文件夹，可以直接在/source/_posts目录下创建一个和博客文章同名的文件夹，放入对应的图片就行。

小结

到目前为止只是对博客的文本、代码、图片做了插入，还有很多不完善的地方。如博客整理归纳的问题、博客排版问题、代码的美观问题、公式的插入等等细节还需要通过多编写博客进行完善。

摘要

因为学生期间一直没有养成很好的总结归纳的习惯，所以打算借着这段比较充裕的时间来谢谢博客提高自己。不算前面按着视频教程写的那一篇，这一篇才算的上是我真正意义上的第一篇博客。打算先按着Essential C++的习题来写一系列博客。期间还要进行UI Automation的学习，另外学习的过程中可能会碰到其他一些实战的内容，这些都可以用博客来记录。既然已经打算了走上程序员这条路，作为一个大龄小白前方的路肯定的坎坷的，希望自己能在技术这条路上能够坚持下去。

语言环境

编译器：VSCode
语言：C++

习题1.4

要点：因为比较简单，所以我就简单地把自己写的代码拷上来。

习题1.5

题目：查阅相关书籍。
要点：
1、注意使用C-style字符串需要包含cstring头文件，另外求一个cstring字符串的长度函数为sizeof(char* )和strlen(char* )
2、sizeof()，strlen()两个函数之间的区别：是否把最后的结束标志也加上去。strlen()不加，而sizeof()加。
详细的区别请见：https://www.cnblogs.com/wanghao-boke/p/11566633.html
3、C语言中没有字符串，于是使用字符串数组来代替字符串。在C++中，不建议使用C-style的字符串。

习题1.6

题目：查阅相关书籍。
要点：
1、注意不能用cin>>vector或cin>>array的方式来输入容器的元素，具体容器的operater>>重载预算符需要查阅相关资料。
另外，英文中的array可能不是指array容器，有可能指代的是数组。
2、array和vector两者之间的区别就如同C-style字符串和string之间的区别，C++中不建议使用array。

习题1.7

题目：查阅相关书籍。
要点：
1、inputFile>>word在正确读取一行信息时返回true，读取错误时返回false。因此用while(inputfile>>word)来读取所有行的信息
2、ifstream类型没有重载operator<<运算符，同理ofstream没有重载operator>>运算符。
3、相对路径的问题没有解决
